# Lab 3: Build a Portfolio Dashboard with Streamlit (AI-Generated)

**Time:** ~10 minutes  
**CoCo Feature:** Streamlit App Generation & Deployment via AI  
**Learning Objective:** Use CoCo CLI to generate a fully working Streamlit in Snowflake portfolio dashboard from a plain English description and deploy it to your personal schema.

---

## Background

Internal tooling teams at BlackRock regularly build dashboards for portfolio monitoring. CoCo can generate a fully functional Streamlit app from a natural language description — letting developers skip boilerplate entirely and focus on business logic.

---

## Prerequisites

Complete **Lab 1** and **Lab 2** — your personal schema must contain all 4 tables with data.

If your CoCo session was closed, restart and re-set context:

```bash
cortex --connection <connection_name>
```

Then at the prompt:

```
/sql SET hol_database = '<YOUR DATABASE FOR HOL>'
/sql SET hol_warehouse = '<YOUR WAREHOUSE FOR HOL>'
/sql SET my_schema = '<YOUR SCHEMA FOR THE HOL>'
/sql USE DATABASE IDENTIFIER($hol_database)
/sql USE SCHEMA IDENTIFIER($my_schema)
/sql USE WAREHOUSE IDENTIFIER($hol_warehouse)
```

---

## Step 1 — Generate the Streamlit App

At the CoCo prompt, type:

### CoCo Prompt — Generate the App

```
Create a Streamlit in Snowflake app for a Portfolio Holdings Dashboard.
Save it to portfolio_dashboard.py.

The app should use the active Snowflake session — do not hardcode any database,
schema, or warehouse names.

The tables are in the current schema:
- PORTFOLIO_HOLDINGS (portfolio_id, asset_id, ticker, asset_class, sector, quantity, market_value_usd, as_of_date)
- ASSET_PRICES (asset_id, ticker, price_date, close_price, volume)

Build the following layout:

1. Page title: "BlackRock Portfolio Dashboard" (wide mode)

2. Sidebar dropdown (st.selectbox) to select a portfolio_id,
   populated dynamically from PORTFOLIO_HOLDINGS

3. A KPI row with 3 st.metric cards:
   - Total Market Value (formatted as $X.XXM or $X.XXB)
   - Number of Holdings (distinct ticker count)
   - Number of Sectors (distinct sector count)
   All filtered by the selected portfolio_id.

4. A bar chart showing sector allocation by market value for the selected portfolio.

5. A table showing the top 10 holdings by market_value_usd
   with columns: ticker, sector, asset_class, quantity, market_value_usd

Use st.cache_data to cache query results.
```

CoCo will generate and save `portfolio_dashboard.py` in your current directory.

## Step 2 — Review the Generated Code

At the CoCo prompt, ask:

```
Show me the key sections of portfolio_dashboard.py — specifically:
1. How the Snowflake session is obtained
2. Where st.cache_data is applied
3. How the portfolio dropdown is populated
```

CoCo will explain the generated code inline without you needing to open the file manually.

---

## Step 3 — Iterate: Add a Price History Tab

At the CoCo prompt:

### CoCo Prompt — Add a Second Tab

```
Refactor portfolio_dashboard.py to use st.tabs with two tabs:

Tab 1 — "Holdings Overview": keep everything that is already there

Tab 2 — "Price History":
   - A dropdown to pick a ticker from the selected portfolio
   - A line chart showing 30-day close_price history from ASSET_PRICES
   - A st.metric showing the 30-day return % (last close vs close 30 days ago)

Save the updated file.
```

---

## Step 4 — Deploy to Your Personal Schema

At the CoCo prompt:

```
Deploy portfolio_dashboard.py as a Streamlit app to my personal Snowflake schema.
The app should be named PORTFOLIO_DASHBOARD.
Use the current session context for the database and schema — do not hardcode names.
```

CoCo will generate and execute the `CREATE STREAMLIT` statement, deploying the app to:

```
<HOL_DATABASE>.<YOUR_USERNAME>_hol.PORTFOLIO_DASHBOARD
```

---

## Step 5 — Validate the App

Open Snowsight → **Streamlit** section → find your `PORTFOLIO_DASHBOARD` app and open it.

Verify:
- [ ] Portfolio dropdown switches between `BLK_CORE_EQ`, `BLK_GROWTH`, `BLK_VALUE`
- [ ] KPI cards update on portfolio selection
- [ ] Sector bar chart reflects the correct distribution
- [ ] Price History tab shows a 30-day line chart for a selected ticker

---

## Step 6 — (Optional / Stretch Goal)

At the CoCo prompt:

```
Add a third tab "Trade Activity" to portfolio_dashboard.py that shows:
- A bar chart of buy vs sell trade count per ticker for the selected portfolio
  using TRADE_HISTORY
- A summary table: ticker, buy_count, sell_count, net_quantity (buys minus sells)
Save and redeploy.
```
